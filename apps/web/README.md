# @chartroom/web

Next.js 16 App Router web app for Chartroom. Two-panel layout with chat-driven chart generation.

## Running

```bash
bun run dev    # Start dev server on localhost:3000
bun run build  # Production build
```

## Layout

- **Left panel** ŌĆö Chat interface with CSV/Excel upload, model tier selector (fast/standard/power), multi-turn conversation with auto-send after tool calls
- **Right panel** ŌĆö Chart preview, data table, spec editor (visual + JSON via CodeMirror), theme selector, PNG/SVG export

## AI layer

Uses Vercel AI SDK (`streamText`, `useChat`) with OpenRouter:
- `render_chart` ŌĆö client-side tool (no `execute`), renders via `renderVegaLite`, captures PNG
- `lookup_docs` ŌĆö server-side tool, queries core docs
- Auto-send logic: up to 5 auto-sends after tool calls complete

Model tiers configured in `lib/agent/models.ts`:
- Fast: `qwen/qwen3.5-35b-a3b`
- Standard: `qwen/qwen3.5-122b-a10b`
- Power: `qwen/qwen3.5-397b-a17b`
- Overridable via `NEXT_PUBLIC_MODEL_FAST`, `NEXT_PUBLIC_MODEL_MID`, `NEXT_PUBLIC_MODEL_POWER` env vars

## Visual spec editor

Accordion-based UI for editing Vega-Lite specs without writing JSON:
- **Mark editor** ŌĆö 13 mark types (bar, line, area, point, rect, rule, text, tick, arc, boxplot, circle, square, trail) + tooltip toggle
- **Encoding editor** ŌĆö 11 channels (x, y, color, size, shape, opacity, theta, radius, text, detail, order) with type and aggregate selectors
- **Transform editor** ŌĆö 6 transform types (filter, calculate, fold, aggregate, bin, window) with 10 aggregate and 14 window operations
- **Layer editor** ŌĆö Multi-layer composition with per-layer mark/encoding editing
- **Computed fields** ŌĆö Auto-extracts fields generated by transforms

## Auth & billing (optional)

Controlled by `NEXT_PUBLIC_AUTH_ENABLED` (default: `false`). When disabled, the app works without any authentication ŌĆö ideal for self-hosting.

When enabled:
- **OAuth** via Supabase (Google, GitHub providers)
- **Database** ŌĆö PostgreSQL via Drizzle ORM (profiles, usage logs, payments)
- **Stripe payments** ŌĆö Credit top-ups ($5, $10, $25) via checkout sessions + webhooks
- **Usage tracking** ŌĆö Per-token cost deduction with configurable markup multiplier
- **Free credits** ŌĆö Configurable amount granted on first sign-up (default: $1.00)

## API routes

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/chat` | POST | Streaming chat with tool execution + balance checking |
| `/api/stripe/checkout` | POST | Create Stripe checkout session |
| `/api/stripe/webhook` | POST | Handle Stripe payment completion |
| `/api/user/balance` | GET | Fetch current user balance |
| `/auth/callback` | GET | Supabase OAuth callback + profile creation |

## Environment variables

Copy `apps/web/.env.example` to `apps/web/.env.local` and configure:

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENROUTER_API_KEY` | Yes | OpenRouter API key |
| `NEXT_PUBLIC_AUTH_ENABLED` | No | Enable auth + billing (default: false) |
| `NEXT_PUBLIC_MODEL_FAST` | No | Fast tier model ID |
| `NEXT_PUBLIC_MODEL_MID` | No | Standard tier model ID |
| `NEXT_PUBLIC_MODEL_POWER` | No | Power tier model ID |
| `NEXT_PUBLIC_SUPABASE_URL` | If auth | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | If auth | Supabase anon key |
| `DATABASE_URL` | If auth | PostgreSQL connection string |
| `STRIPE_SECRET_KEY` | If auth | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | If auth | Stripe webhook signing secret |
| `FREE_CREDITS_USD` | No | Free credits on sign-up (default: 1.00) |
| `MARKUP_MULTIPLIER` | No | Cost markup multiplier (default: 1.3) |
